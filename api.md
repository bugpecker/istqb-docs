1. طراحی API با کیفیت

کلیدواژه‌ها:

GET /users

4. بررسی عملکرد API

5. نسخه‌بندی API

۳۶. در تست رابط کاربری (UI)، چگونه می‌توان از بروز خطاهای ظاهری جلوگیری کرد؟

پاسخ:
با استفاده از تست‌های خودکار رابط کاربری (UI Automation Tests) + بررسی طراحی واکنش‌گرا (Responsive Design) + تست در مرورگرها و دستگاه‌های مختلف.

توضیح:
خطاهای ظاهری معمولاً به خاطر تفاوت مرورگر یا اندازه صفحه رخ می‌دهند.

مثال:
با ابزارهایی مثل Selenium (سلنیوم)، چک کنیم دکمه‌ها در موبایل هم درست نمایش داده شوند.

۳۷. در تست UI، چه تفاوت‌هایی بین تست دستی و تست خودکار وجود دارد؟

پاسخ:

تست دستی: توسط انسان انجام می‌شود
تست خودکار: توسط اسکریپت اجرا می‌شود

توضیح:
تست دستی برای کشف مشکلات ظاهری بهتر است، تست خودکار برای تکرار سریع.

مثال:

دستی: بررسی زیبایی صفحه
خودکار: چک کردن اینکه فرم ثبت‌نام کار می‌کند
۳۸. چگونه می‌توان تست‌های UI را به صورت خودکار مدیریت کرد؟

پاسخ:
با استفاده از فریم‌ورک‌های تست (Test Frameworks / چارچوب‌های تست) و ابزارهای CI/CD (یکپارچه‌سازی و تحویل مداوم).

توضیح:
این ابزارها تست‌ها را به صورت خودکار اجرا و گزارش می‌دهند.

مثال:
اجرای تست Selenium در Jenkins (جنکینز) بعد از هر تغییر کد.

۳۹. در صورت تغییر رابط کاربری، چگونه باید تست‌ها را به‌روزرسانی کرد؟

پاسخ:
با استفاده از الگوی طراحی Page Object Model (مدل شیء صفحه) برای کاهش وابستگی تست‌ها به UI.

توضیح:
اگر UI تغییر کند، فقط یک بخش اصلاح می‌شود نه کل تست‌ها.

مثال:
اگر دکمه Login تغییر کند، فقط فایل مربوط به آن صفحه اصلاح می‌شود.

۱. چگونه API مقیاس‌پذیر طراحی می‌شود؟

پاسخ امتحانی:
با استفاده از کش (Cache / حافظه موقت)، لود بالانسر (Load Balancer / توزیع‌کننده بار) و طراحی بدون وضعیت (Stateless / بدون حالت).

توضیح کوتاه:
این روش‌ها باعث افزایش توان پاسخ‌گویی سیستم در تعداد کاربر بالا می‌شوند.

مثال:
استفاده از Redis (ردیس) برای ذخیره پاسخ‌های پرتکرار.

۲. چگونه امنیت API افزایش می‌یابد؟

پاسخ امتحانی:
با استفاده از احراز هویت (Authentication / هویت‌سنجی)، مجوزدهی (Authorization / سطح دسترسی) و WAF (فایروال برنامه وب).

توضیح:
برای جلوگیری از حملاتی مثل Injection (تزریق).

مثال:
استفاده از JWT (توکن امنیتی) برای کاربران.

۳. تفاوت REST و GraphQL چیست؟

پاسخ امتحانی:
REST ساده و مبتنی بر endpoint است، GraphQL انعطاف‌پذیر و امکان دریافت دقیق داده را می‌دهد.

توضیح:
GraphQL داده اضافی برنمی‌گرداند.

مثال:
در GraphQL فقط فیلد name گرفته می‌شود نه کل آبجکت.

۴. مانیتورینگ API چگونه انجام می‌شود؟

پاسخ امتحانی:
با لاگ‌گیری (Logging / ثبت رویداد) و پایش (Monitoring / نظارت).

توضیح:
برای بررسی عملکرد و خطاها.

مثال:
بررسی latency (تاخیر) و error rate (نرخ خطا).
